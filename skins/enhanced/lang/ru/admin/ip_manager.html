LANG_IP_MANAGER=Менеджер IP LANG_ADD_IP=Добавить IP LANG_NETMASK=Маска сети LANG_IPV6_MASK=Для IPv6 IPs, используйте /mask, напр.: /64 LANG_DETAILS_FOR=Детали для LANG_STATUS=Статус LANG_NETMASK=Маска сети LANG_USER_VALUE=Пользователь(и) LANG_NAMESERVER=Неймсервер LANG_IP_PURPOSE=Основным назначением этой функции является связка IP-адресов LAN и внешних IP-адресов,
и/или привязать IP-адреса IPv6 к IPv4 IP-адресам. LANG_LAN_EXPLANATION=LAN: Просмотр сведений о внешнем IP. Связать внутренний IP на внешний IP.
Выберите только Apache, не выбирайте DNS. Добавить пользователей к внешнему IP LANG_IPV6_EXPLANATION=IPv6: Просмотр сведений про IPv4 IP. Связь IPv6 с IPv4. Отметьте оба DNS и Apache. LANG_LINK_OPTIONS_EXPLANATION=Добавление связанного IP к текущему IP добавит связанный IP к зоне где есть текущий IP.
Связь IP с dns создаст дубликат записи A/AAAA records для текущего IP со связанным IP.
Связь IP с apache добавит связанный IP в VirtualHost с текущим IP. LANG_MULTI_IP_RELATION=Эта функция, не зависит от системы Multi-IP, но они могут быть использованы вместе.
Например у пользователя есть 2 IP c Multi-IP системой, и каждый и этих IP имеет присоединенные IP. LANG_LINK_NOT_RECURSIVE=Ссылки не рекурсивные, так что вам не стоит беспокоиться про бесконечный цикл. LANG_APPLY_TO_DOMAINS=Опция "Применить к существующим доменам" выполнит перезапись httpd.conf и вставит записи A/AAAA в локальную DNS зону.
Отключение этой опции не затронет файлы dns или httpd.conf, но будущие перезаписи httpd.cont будут с добавлением связанных IP.
